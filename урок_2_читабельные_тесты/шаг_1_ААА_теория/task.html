ААА - теория

ААА! Вот уже второй урок - круто!

Тесты мы запускать уже умеем. Правда написать правильно работающий тест - это ещё пол дела, тесты должны быть понятно написанными. Ведь в большом проекте их колличество может достигать десятки тысяч тестов. А бывает и так, что в одном только тестовом файле тестовых методов больше сотни...

Для того, чтобы облегчить жизнь себе и другим разработчикам, читающим ваш код, принято одинаково оформлять тесты. В том числе соблюдать одинаковую структуру у тестового файла.

Есть популярный паттерн ААА, призванный помочь одинаково структурировать тела тестовых методов. Каждая А в ААА, это одна из 3 частей тела тестового метода:

  • Arrange - подготовка тестовых данных
  • Act - запуск тестируемого действия
  • Assert - проверка, что всё прошло, как мы и думали

Вообще, любой тест состоит из этих 3 этапов. Но другое дело сделать тест читабельнее, подчеркнув где какая из 3 частей. Даже если просто выделить каждую из 3 частей дополнительным переносом на новую строчку или комментарием типа // Arrange // Act // Assert, то читать уже гораздо проще.

В файле SimpleTest показан простой пример для наглядности, как может выглядеть тест, структурированный по паттерну ААА.

Нажимайте на кнопку проверки: SimpleTest пройдёт и можно двигать дальше. Ничего добавлять даже не нужно.
